# 直到一万点 · 投资仪表盘

基于雪球博主 **@直到一万点** 的投资方法论，构建的 **方法论驱动的主动信号系统**。

**核心理念**: 方法论不是静态文档，而是信号引擎。网页不是展示页，而是操作终端。

---

## 一、项目定位

这个系统的目的是把"直到一万点"15章投资方法论**编程化**，每日自动：

1. **选股评分** — 按方法论权重对全A股打分排序
2. **信号检测** — 检测买入/卖出/板块切换/仓位调整信号
3. **持仓跟踪** — 针对你的持仓给出每日策略建议
4. **自选监控** — 自选股评分追踪和买入就绪提醒

所有信号和策略背后都有方法论章节来源，不是黑箱。

## 二、方法论核心（来自 @直到一万点）

### 评分体系（总分100）

| 维度 | 满分 | 来源章节 |
|------|------|----------|
| 多头排列 | 40 | 04-个股筛选体系 |
| 突破新高 | 25 | 05-买入时机与买点体系 |
| 趋势加速 | 15 | 09-技术分析工具箱 |
| 量价配合 | 10 | 05-买入时机 |
| 基本面代理 | 10 | 04-个股筛选体系 |

### 买入信号（5种）

| 信号 | 条件 | 强度 | 来源 |
|------|------|------|------|
| 突破买入 | 当日突破20日新高 + 放量(量比>2) | 强 🟢 | 05章 模式二 |
| 多头启动 | 刚形成完全多头排列（昨天还不是） | 强 🟢 | 05章 模式一 |
| 均线低吸 | 上升趋势 + 均线上方 + 评分≥60 | 中 🔵 | 05章 均线低吸法 |
| 中枢突破 | 突破平台 + 温和放量 + 评分≥75 | 中 🔵 | 05章 振荡中枢上车 |
| 趋势加速 | 刚进入加速状态（5日>10日>20日） | 弱 🟡 | 05章 趋势加速 |

### 卖出信号（4种）

| 信号 | 条件 | 强度 | 来源 |
|------|------|------|------|
| 多头排列消失 | 今天非完全多头 + 昨天是 + 5日转负 | 强制 🔴 | 06章 趋势结束 |
| 高位破位 | 高分股单日跌幅>5% | 强制 🔴 | 06章 高位大阴线 |
| 持续走弱 | 连续走弱 + 非多头排列 | 强烈 🟠 | 06章 不及预期 |
| 加速赶顶 | 连续大涨后今日转跌 | 减仓 🟠 | 06章 主动让利 |

### 仓位管理

- 牛市 + 多头 + 高分 → 积极仓位，可加仓
- 熊市 → 防御仓位或空仓
- 震荡市 → 中性仓位，只做确定性机会

## 四、数据库表结构

数据库路径: `data/screener/screener.db`（SQLite，WAL模式）

### 选股数据（每次选股覆盖写入）

| 表名 | 说明 | 主键 |
|------|------|------|
| `index_daily` | 每日指数评估 | (date, name) |
| `sector_daily` | 每日板块排名 | (date, name) |
| `stock_daily` | 每日个股评分 | (date, code, sector) |

### 用户数据（持久保留，不会覆盖）

| 表名 | 说明 |
|------|------|
| `portfolio` | 持仓记录（入场价、日期、止损方式等） |
| `watchlist` | 自选股（目标价、提醒分数、关注理由） |
| `daily_signals` | 每日信号日志（买/卖/板块/仓位） |
| `portfolio_snapshot` | 每日持仓快照（净值曲线数据） |

### 缓存数据

| 表名 | 说明 |
|------|------|
| `kline_cache` | K线数据缓存（4小时TTL） |
| `stock_tracking` | 个股出现历史（用于回溯分析） |
| `screener_runs` | 选股运行记录 |

**⚠️ 重要**: 不要删除或清空 `portfolio`、`watchlist` 表。它们存的是你的投资记录。

## 五、日常使用

### 5.1 每日流水线

```bash
python run_daily.py
```

执行6个步骤：

```
Step 1/6: 运行选股 (API调取全A股K线→评分→排序)
Step 2/6: 生成信号 (用方法论规则检测买卖/板块/仓位信号)
Step 3/6: 导出持仓数据 (生成portfolio.json给仪表盘)
Step 4/6: 复制选股结果 (latest.json → data/web/)
Step 5/6: 准备Web数据 (雪球分析+方法论数据)
Step 6/6: 重建KB索引 (全文搜索索引)
```

**运行频率**: 每日收盘后（15:30以后）运行一次。

### 5.2 启动仪表盘

```bash
python run_web.py
```

浏览器自动打开 `http://127.0.0.1:8080`。

### 5.3 仪表盘5个标签页

| 标签 | 快捷键 | 功能 |
|------|--------|------|
| **首页·信号** | `1` | 市场状态 + 买卖/板块/仓位信号面板 + 持仓概览 |
| **持仓管理** | `2` | 添加/编辑/平仓持仓，查看策略建议 |
| **自选股** | `3` | 管理自选股，追踪评分和买入就绪状态 |
| **每日选股** | `4` | 选股结果、板块排名、指数趋势 |
| **投资体系** | `5` | 方法论全文浏览 + 搜索 |

### 5.4 添加你的持仓

1. 打开仪表盘 → 标签页2「持仓管理」
2. 点击「+ 添加持仓」
3. 填写股票代码（如 `sh688256`）、名称、入场价、入场日期
4. 选择止损方式（MA10 / MA20 / 固定价）
5. 保存后，返回首页查看策略建议

### 5.5 添加自选股

1. 标签页3「自选股」→ 「+ 添加自选」
2. 填入代码和目标买入价、提醒分数
3. 每日选股后，自选股会自动显示当前评分和是否就绪

## 六、数据维护指南

### 6.1 更新雪球帖子

当 @直到一万点 发布新帖子时：

1. 将帖子数据追加到 `data/xueqiu_posts/xueqiu_posts/backup_5188.json`
2. 运行 `python web/data_prep.py` 重新生成分析数据
3. 新方法论洞察 → 更新 `knowledge_base/` 中对应的 `.md` 文件
4. 运行 `python run_daily.py` 重建KB索引

### 6.2 更新公众号文章

1. 将文章内容保存到 `data/wechat_articles/`
2. 更新 `knowledge_base/appendix/A-完整文章索引.md`
3. 如有新方法论内容，更新对应章节 `.md` 文件
4. 运行 `python knowledge/builder.py` 重建索引

### 6.3 更新方法论规则

当你从帖子/文章中提取到新的投资规则时：

1. **先在 `knowledge_base/` 更新对应的 `.md` 文件** — 这是方法论的唯一权威来源
2. 如果规则可编程化，在 `screener/signals.py` 添加对应的信号检测逻辑
3. 在信号函数中标注来源章节（如 `'05-买入时机: 新模式X'`）
4. 如果影响评分，在 `screener/scoring.py` 调整
5. 运行 `python run_daily.py` 验证

**原则**: 方法论文档 → 信号逻辑 → 仪表盘展示。永远是这个顺序。

### 6.4 更新 @直到一万点 的持仓

如果他在帖子里透露了持仓变动：

1. 在仪表盘标签页2中添加/编辑持仓
2. 在入场理由里注明「来自帖子 YYYY-MM-DD」
3. 或者直接在数据库操作（保留来源记录）

### 6.5 数据库备份

```bash
# 复制整个数据库文件即可
copy D:\stock\data\screener\screener.db D:\backup\screener_20260504.db
```

## 七、设计原则（修改代码前必读）

### 7.1 方法论驱动，不是数据驱动

- 每个信号、每个评分维度都必须追溯到方法论的某个章节
- 不要在 `signals.py` 里添加"看起来合理"但没有方法论支持的信号
- 看到新规则 → 先更新 `knowledge_base/` → 再改代码

### 7.2 数据库是永久资产

- `portfolio`、`watchlist` 表包含你的投资记录，**绝对不能丢失**
- 新增表用 `CREATE TABLE IF NOT EXISTS`，不能删除重建已有表
- 选股数据（`stock_daily`、`sector_daily` 等）是按日期覆盖的，历史数据在 `daily_*.json` 文件中有备份

### 7.3 简单优先

- 不引入新框架（当前是纯 Python + HTML + Chart.js）
- 不做前端构建工具链（一个 `index.html` 直接可用）
- 不做数据库ORM（直接 SQLite + 原生SQL）
- 评分权重、板块定义、指数列表统一在 `config.py` 维护

### 7.4 信号不重复

- 同一股票在不同板块中出现时，信号引擎会自动去重（取最高评分那条）
- 每日运行前清除当日旧信号再写入

### 7.5 中文优先

- 所有文档、注释、仪表盘界面使用中文
- 数据库字段名使用英文（兼容性考虑）
- 信号名称、策略建议使用中文

## 十、常见问题

**Q: 为什么持仓的「当前价格」显示为0？**
A: 当天还没运行 `run_daily.py`。选股数据只在每日运行后更新。

**Q: 自选股显示「等待(X/Y)」是什么意思？**
A: 当前评分(X)还没达到你设定的提醒分数(Y)。它会在每日选股后自动更新。

**Q: 如何修改评分权重？**
A: 编辑 `config.py` 中的 `SCORE_WEIGHTS`，然后运行 `run_daily.py`。

**Q: 如何添加新的板块？**
A: 在 `config.py` 的 `SECTORS` 字典中添加板块名和代表性股票代码列表。

**Q: 数据库可以手动编辑吗？**
A: 可以。推荐使用 [DB Browser for SQLite](https://sqlitebrowser.org/) 打开 `data/screener/screener.db`。
